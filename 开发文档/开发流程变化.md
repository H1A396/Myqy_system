## 第三阶段 IDT和异常处理

```
引用-IDT和异常处理.md
```

遇到的问题
```
EXCEPTIoN: General Protection Fault
(Error Code:
16)
System halted
```

## 问题根因
pic_remap() 初始化时把所有 IRQ 都取消了屏蔽 （写 0x00 而不是 0xFF ），导致 sti 之后定时器、键盘等硬件中断立即涌入。虽然注册了处理函数，但某些硬件中断在无实际驱动的情况下触发了 #GP。
## 修复内容
文件 改动 pic.c pic_remap() 最后改为屏蔽所有 IRQ（ 0xFF ）；新增 pic_unmask() / pic_mask() 函数 pic.h 新增 pic_unmask() / pic_mask() 声明 idt.c irq_handler 中 EOI 逻辑修正：只有 IRQ8-15（向量 40-47）才向从 PIC 发 EOI；tick 计数改为检查向量 32

## 当前状态
- 系统启动后显示 5 行信息，然后稳定运行不崩溃
- 所有 IRQ 已屏蔽，需要时可通过 pic_unmask(irq) 按需开启
- 异常处理框架就绪：任何 CPU 异常都会被捕获并显示错误信息
